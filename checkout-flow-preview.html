One row per worktree, 2-line layout: mono path on top, meta line below with a muted status dot, the branch it currently holds, and ahead/behind/dirty tally.
  • Selected row shows a left accent bar and filled ◆ marker.
  • "New worktree" is a single CTA button below the list, pre-filled with a sanitized default derived from the PR's branch name.
  • Main worktree is hidden behind a toggle. Clicking checkout into main overwrites the user's current branch — too dangerous to be a default-visible choice.
  • Single primary "Check out here →" button lives in the footer; Cancel to its left.

A status strip at the top of the drawer summarizes what's happening in one line: spinner · Staging branch · target path.
  • Below it, a terminal log streams the actual git commands and their output — no more guessing what the app is doing.
  • Accent-colored $ prefix, muted output lines, blinking caret on the last line while running.
  • The footer buttons are disabled until the run finishes.

Nothing disappears. The log stays, the status strip turns green, and four icon+label chips appear as a single row at the bottom of the drawer. The drawer stays open until the user dismisses it.
  • Each icon is a hand-drawn SVG, not an emoji — they stay consistent across OS theme changes.
  • Buttons are even-width in a 4-column grid; label truncates rather than wrapping.
  • Clicking Checkout again on another PR replaces the drawer's content.

What went wrong
A directory already exists at the target path. Remove it, or pick a different worktree name.
Errors don't collapse the log — the raw stderr stays visible. A plain-English "What went wrong" block below it interprets common git failures (path exists, permission denied, branch checked out elsewhere) so users don't need to read git-ese.
